    Integration

    Android Native Apps#

    Overview#

    Chyme SDK is an android plugin that can be used to add Chatbots to any android application.

    Required

    Add the following dependencies to app build.gradle file 

    implementation 'de.hdodenhof:circleimageview:2.2.0'
implementation 'com.github.dmytrodanylyk.circular-progress-button:library:1.1.3'
implementation 'com.orhanobut:dialogplus:1.11@aar'
implementation 'com.android.support:customtabs:27.0.2'
implementation ('com.github.bumptech.glide:glide:3.8.0') {
 exclude module: 'okio'
}
implementation('io.socket:socket.io-client:1.0.0') {
 exclude group: 'org.json', module: 'json'
 exclude module: 'okio'
}
implementation 'com.unvired.android-sdk:3.0.187' (Note: always use latest available from maven)

    Add following code in app manifext.xml file

    <!-- ==================== Chat Window Screens ==================== -->

    <activity
android:name="com.unvired.chymesdk.activities.ChatWindowActivity"
android:theme="@style/AppTheme" />
<activity
android:name="com.unvired.chymesdk.activities.ChatLoginActivity"
android:theme="@style/AppTheme" />

<activity
android:name="com.unvired.chymesdk.activities.AudioViewActivity"
android:theme="@style/AppTheme" />

<activity
android:name="com.unvired.chymesdk.activities.ImageViewActivity"
android:theme="@style/AppTheme" />

<activity
android:name="com.unvired.chymesdk.activities.WebViewActivity"
android:theme="@style/AppTheme" />

    Usage

    Method 1: Using "AssitantFAB"

    Include AssistantFAB view in xml layout wherever needed.

    <com.unvired.chymesdk.custom.AssistantFAB
     android:layout_width="wrap_content"
     android:layout_height="wrap_content"
     android:layout_margin="16dp"
     android:layout_gravity="bottom|end"
     app:json_config="@string/assistant_config" />
    - string/assistant_config is the ChatBot configuration in JSON format.

    Method 2: Start activity programmatically 

    Intent intent = new Intent(this, ChatLoginActivity.class);
intent.putExtra(AssistantHelper.CONFIG, getString(R.string.assistant_config));
intent.putExtra(KEY, VALUE);
startActivity(intent);
    - string/assistant_config is the ChatBot configuration in JSON format. - Pass values using KEY-VALUE pair if needed.

    REST API#

    Overview#

    Version information#

    Version : 0.0.9

    URI scheme#

    Host: app.sandbox.chyme.io BasePath : /API Schemes: HTTPS

    Tags#

    Files: Operations about Files

    Messages: Operations about Messages

    Paths#

    Upload a file to the Chyme server to attach to a message later#

    POST /files

    Description#

    Uploads a file to the Chyme server. If a mediaId is specified then the file is saved on Chyme with that id or a new mediaId is created and returned. The file can be attached with a message by using the returned mediaId and posting it along with the message data to the SEND_MESSAGE API.

    Parameters#

    Type Name Description Schema Default
    FormData filename
    required    		 The file to upload file
    FormData mediaId
    optional    		 Specify the unique mediaId if you require Chyme to cache the file with this id. If not specified a new mediaId will be created. string

    Responses#

    HTTP Code Description Schema
    200 OK UploadResponse

    Consumes#

    multipart/form-data

    Produces#

    application/json

    Tags#

    • Files

    Security#

    Type Name Scopes
    basic Default
    Example HTTP response#

    Response 200 

    {
  "application/json" : {
    "mediaId" : "c99ae254-b192-4f01-9868-df1493effe80"
  }
}

    Download the file specified by the mediaId from Chyme server.#

    GET /files/{mediaId}

    Description#

    Download the file specified by the mediaId from the Chyme server.

    Parameters#

    Type Name Description Schema Default
    Path mediaId
    required    		 mediaId string

    Responses#

    HTTP Code Description Schema
    200 Returns the file in the body No Content

    Tags#

    • Files

    Security#

    Type Name Scopes
    basic Default

    Send a message to a user, group, channel or assistant.#

    POST /messages

    Description#

    Send a message to a user, group, channel or assistant. Also attach previously uploaded files to the message.

    Parameters#

    Type Name Description Schema Default
    FormData assistant
    optional    		 The name of the assistant to post the message as. This parameter is required ONLY if an external system is posting the message via a WebHook URL. string
    FormData conversationId
    optional    		 If the message is being posted as part of an existing conversation, provide the conversationId. Not providing this will default to creating a new conversation. string
    FormData language
    optional    		 The language of the included message. Not required if English, for other languages set it to the 2 character ISO language code string "en"
    FormData mediaId
    optional    		 If a file has already been uploaded to the Chyme server and needs to be attached to this message, provide the mediaId that was returned in the UPLOAD_FILE api. string
    FormData message
    required    		 The message to send string
    FormData recipient
    required    		 The message recipient. The recipient can be the @name of a normal user, an assistant, a group or a channel. string

    Responses#

    HTTP Code Description Schema
    200 JSON output: recipient - the recipient to whom the message was sent successfully conversationId - The conversation this message is a part of messageId - Unique identifier of this message <SendMessage> array

    Consumes#

    application/x-www-form-urlencoded

    Produces#

    application/json

    Tags#

    • Messages

    Security#

    Type Name Scopes
    basic Default
    Example HTTP response#

    Response 200 

    {
  "application/json" : [ {
    "recipient" : "adam",
    "conversationId" : "f728967b-c063-4117-a35b-d5f5979e5da9",
    "messageId" : "f728967b-c063-4117-a35b-d5f5979e5da9"
  } ]
}

    Retrieve the replies for a message posted to an assistant.#

    GET /messages/{messageId}/replies

    Description#

    Retrieve the replies for a message posted to an assistant. Pass the messageId whose replies need to be retrieved. One ore more replies may be returned in the response. All replies that have not yet been delivered will be returned.

    Parameters#

    Type Name Description Schema Default
    Path messageId
    required    		 messageId string

    Responses#

    HTTP Code Description Schema
    200 Returns the available replies to the specified messageId <MessageDetails> array

    Produces#

    application/json

    Tags#

    • Messages

    Security#

    Type Name Scopes
    basic Default
    Example HTTP response#

    Response 200

    {
  "application/json" : [ {
    "conversationId" : "c2b90edb-861c-4ec3-9316-51f33f53adbd",
    "referenceId" : "",
    "messageText" : "Doing great buddy,",
    "contentText" : "",
    "longitude" : "",
    "latitude" : "",
    "timeAtServer" : 1502273147127,
    "commandName" : "",
    "actions" : "",
    }]
}
